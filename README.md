# IT Job Search Engine

A hybrid job search engine built on real scraped data from Egyptian job boards (Indeed, LinkedIn, Wuzzuf). Users can search by keyword,description,tools etc(lazy search) and get ranked results using a combination of keyword matching and semantic similarity.

---

## How it works

Queries are processed through two retrieval systems that run in parallel and combine their scores:

- **BM25** — keyword matching with lemmatization and stop word removal.
- **Semantic search** — sentence embeddings via `all-MiniLM-L6-v2` + FAISS vector index.

Final score = `0.4 × BM25 score + 0.6 × semantic score`

## Usage

### 1 — Provide your data
Place your cleaned jobs CSV at `data/WorkingData.csv` with these columns:
```
url | title | company | job_type | employment_type | description | requirements
```

### 2 — Build the indexes
```bash
python retrieval/build_index.py
```
This runs once (~3–5 minutes for embeddings) and saves:
- `bm25_title.pkl` — BM25 index on job titles
- `bm25_full.pkl` — BM25 index on full search documents
- `job_vectors.npy` — sentence embedding vectors
- `jobs.faiss` — FAISS vector index
- `data/jobs.pkl` — dataframe for result lookup

### 3 — Search
```bash
python retrieval/search.py
```
Or import the search function directly:
```python
from retrieval.search import search

results = search("python backend engineer", top_k=10)
print(results)
```

### 4 — Update indexes after new data
```bash
python retrieval/build_index.py
```
Just re-run build_index whenever your data changes.
